Extension

An extension is a code unit that is run as a part of a server. Unlike services, extensions don't expose RPC endpoints. Instead, the extension code is executed in one or both ways:

  • Provide objects to other extensions/services.
  • Provide global initialisation for a server.

Providing an Instance of an Extension-Specific Type‚Äã

Let's say we want to provide a direct API for a Postgres database. First define an extension with its input type:

// extension.cue
import "namespacelabs.dev/foundation/std/fn"

$providerProto: inputs.#Proto & {
  source: "provider.proto"
}

extension: fn.#Extension & {
  provides: {
    Database: {
      type: $providerProto.types.Database

      availableIn: {
        go: {
          // This is a Go-specific type.
          package: "github.com/jackc/pgx/v4/pgxpool"
          type:    "*Pool"
        }
      }
    }
  }
}
// provider.proto

message Database {
    string name = 1;
    foundation.std.types.Resource schema_file = 2;
}

Namespace generates a Provide interface that needs to be implemented. Example:

// deps.fn.go

type ExtensionDeps struct {
  // Here may be objects of types provided by extensions.
}

type _checkProvideDatabase func(context.Context, string, *Database, ExtensionDeps) (*pgxpool.Pool, error)

var _ _checkProvideDatabase = ProvideDatabase

Implement the extension logic:

// my_file.go

func ProvideDatabase(ctx context.Context, caller string, db *Database, deps ExtensionDeps) (*pgxpool.Pool, error) {
  ...

  return conn, nil
}

Done! The extension-provide typed (*pgxpool.Pool in this case) is available for injection in other Extensions/Services.

Injecting Extension-Provided Objects‚Äã

After an Extension has been defined, it can be injected in other Extensions/Services. Example:

// service.cue
import (
  "namespacelabs.dev/foundation/std/fn"
  "namespacelabs.dev/foundation/std/fn:inputs"
  "namespacelabs.dev/my_package/postgres"
)

service: fn.#Service & {
  instantiate: {
    db: postgres.#Exports.Database & {
      // Filling in the input "Database" proto.
      with: {
        name: "todos"
        schemaFile: inputs.#FromFile & {
          path: "schema.sql"
        }
      }
    }
  }

  ...
}

The provided value can be used via "ServiceDeps"/"ExtensionDeps":

// deps.fn.go

type ServiceDeps struct {
  Db *pgxpool.Pool
}

type checkWireService func(context.Context, *server.Grpc, ServiceDeps)

Global Initialisation‚Äã

If an Extension definition contains hasInitialization: true, its initialization code is executed eagerly, before the server starts serving. An Extension is bundled with and instantiated in a Server if the Server or any of its transitive dependencies import the package with the extension.

Example:

import (
  "namespacelabs.dev/foundation/std/fn"
  "namespacelabs.dev/foundation/std/go/grpc/interceptors"
)

extension: fn.#Extension & {
  hasInitialization: true

  instantiate: {
    "interceptors": interceptors.#Exports.InterceptorRegistration
  }
}

Namespace generates a stub of the Prepare function that needs to be implemented:

type ExtensionDeps struct {
  ...
}

type _checkPrepare func(context.Context, ExtensionDeps) error

var _ _checkPrepare = Prepare
tip

Namespace can bootstrap a new extension for you. Try ns src create extension path/to/extension!
