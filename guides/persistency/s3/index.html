Using S3

After completing these steps, you'll get:

  • A configured AWS S3 client ready to use in your codebase: regardless of environment: dev, prod, etc.
  • Instantiated and policy-configured S3 buckets with the latest policy and authentication recommendations from AWS (e.g. using IAM role-based authorization).
  • In local development and tests, minio-backed S3 buckets for fast ephemeral development.

In the service.cue of the service where you want to use S3, first add an import:

import (
    // ...
    "namespacelabs.dev/foundation/universe/storage/s3"
)

And then instantiate a S3 bucket:

service: fn.#Service & {
    // ...

    instantiate: {
        bundles: s3.#Exports.Bucket & {
            bucketName: "foobar--bundles-bucket"
        }
    }
}

You can optionally specify a region, but you don't usually need to, as Namespace will determine the region from your production deployment.

Note: Namespace does not generate S3 bucket names, you'll need to guarantee that you're using a globally unique name.

After you run ns dev, you'll find a configured instance in your codebase:

func WireService(ctx context.Context, srv server.Registrar, deps ServiceDeps) {
    svc := &Service{
        // deps.Bundles is a S3 client facade, configured with the appropriate
        // credentials, region and bucket.
        bundles: deps.Bundles,
    }
    // ...
}
